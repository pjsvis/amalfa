# Markdown Hygiene Protocol

## 1. Pre-Commit Validation
- Git hooks automatically sanitize line numbers/control chars
- Biome linting enforces consistent formatting (block commit on failure)
- Emergency override: `git commit --no-verify`

## 2. Emergency Recovery
```bash
# Re-sanitize entire corpus (preserves git history)
./github/workflows/sanitize-all.sh

# Nuclear option: Restore from clean branch
 git checkout sanitization-poc -- docs/
```

## 3. Baseline Rules
1. ALL content edits go through sanitization layer
2. NO manual line numbering (`123| ` prefixes forbidden)
3. Biome-formatted diffs required for PRs
