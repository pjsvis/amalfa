### 2. The Component: `src/components/IntelligenceCard.tsx`

This component is designed to be fully self-contained, importing only what is necessary and defining its "Industrial" look through inline-compatible logic or scoped attributes.

```tsx
/**
 * IntelligenceCard Component
 * Principle: Brutalisimo / Self-Contained
 */

interface IntelligenceArtifact {
  headline: string;
  impact_score: number; // 1-10
  hollow_nodes: string[];
  summary_md: string;
  tags: string[];
  timestamp: string;
}

export const IntelligenceCard = ({ data }: { data: IntelligenceArtifact }) => {
  // Score color logic (Industrial palette)
  const getImpactColor = (score: number) => {
    if (score > 7) return "#ff0055"; // High Impact (Critical)
    if (score > 4) return "#00ffcc"; // Medium Impact (Notable)
    return "#888888"; // Low Impact (Noise)
  };

  return (
    <article 
      data-brutal-card 
      style={{
        border: "3px solid #333",
        margin: "1rem 0",
        padding: "1.5rem",
        backgroundColor: "#0a0a0a",
        fontFamily: "monospace",
        position: "relative"
      }}
    >
      {/* Impact Badge */}
      <div style={{
        position: "absolute",
        top: "-15px",
        right: "20px",
        backgroundColor: getImpactColor(data.impact_score),
        color: "#000",
        padding: "2px 8px",
        fontWeight: "bold",
        fontSize: "0.8rem",
        border: "2px solid #333"
      }}>
        IMPACT: {data.impact_score}/10
      </div>

      <header style={{ marginBottom: "1rem", borderBottom: "1px solid #333" }}>
        <h2 style={{ fontSize: "1.2rem", color: "#eee", margin: "0 0 0.5rem 0" }}>
          {data.headline.toUpperCase()}
        </h2>
        <small style={{ color: "#666" }}>{data.timestamp}</small>
      </header>

      <section style={{ color: "#aaa", lineHeight: "1.4" }}>
        {/* We assume a markdown renderer is available globally or passed in */}
        <div data-content-summary>
          {data.summary_md}
        </div>
      </section>

      <footer style={{ marginTop: "1.5rem" }}>
        <div style={{ marginBottom: "0.5rem" }}>
          <strong style={{ color: "#00ffcc", fontSize: "0.8rem" }}>HOLLOW NODES DETECTED:</strong>
          <ul style={{ list-style: "none", padding: 0, display: "flex", flexWrap: "wrap", gap: "10px" }}>
            {data.hollow_nodes.map(node => (
              <li key={node} style={{ color: "#fff", background: "#222", padding: "2px 6px", fontSize: "0.75rem", border: "1px solid #444" }}>
                {node}
              </li>
            ))}
          </ul>
        </div>

        <div style={{ display: "flex", gap: "5px" }}>
          {data.tags.map(tag => (
            <span key={tag} style={{ color: "#666", fontSize: "0.7rem" }}>#{tag}</span>
          ))}
        </div>
      </footer>
    </article>
  );
};

```

### 3. Opinion: The "Two-Step" Workflow in Practice

**Step 1: The TUI Forge (`bin/amalfa-intel`)**
You are in the terminal. You run the fetcher. You see the `mods` summary rendered via `glow`. This is the **Mentation** phase. You decide if today's news is worth a permanent record. If it is, the JSON is saved to the `distilled` folder.

**Step 2: The WUI Gallery (`src/components`)**
The next time you open the Amalfa Dashboard, the SSR server scans `~/.amalfa/intel/distilled/` and maps the files to a list of `IntelligenceCard` components.

**Why this works:**

* **Zero Ingestion Risk:** No vector DB collisions or orphaned edges in your main graph.
* **Workflow Durability (PHI-13):** The cards act as a "Daily Briefing" that survives across sessions.
* **Brutalisimo Compliance:** The card doesn't try to be "pretty"; it tries to be **informative**. The use of monospace, high-contrast badges, and sharp borders mirrors the CLI experience, creating a unified **Persona Stack (Concept-05)**.
